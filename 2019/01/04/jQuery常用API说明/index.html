jQuery常用API使用记录与说明

  • eq(n)：返回被选元素中带有指定索引号的标签元素；

  • addClass(类名)：给选中的元素添加类名；

  • removeClass(类名)：给选中的元素去除类名；

  • hasClass(类名)： 判断指定元素中是否存在对应类名样式，返回布尔类型值，注意不支持查询多个类样式；

  • toggleClass(“类名一 类名二”)：使jq对象样式在两个类名样式之间来回切换；

  • offset()：获取的是当前标签相对于窗口的位移信息(top|left)；

  • position()：获取当前标签元素相对于父元素的位置信息(top|left)；

  • text()：不传参则表示获取元素的文本内容，并拼接返回；传参则表示设置元素的文本内容为传递的参数内容；

  • html()：不传参则表示获取所选中标签元素的第一个标签内容；传参则表示设置节点内容为传递的参数内容；

  • siblings()：获取元素的兄弟元素；

  • filter()：根据自定义规则标准筛选元素，不匹配的元素被删除，匹配条件的元素返回；

    说明：filter的参数可以是一个函数，这个函数用于限定规则并返回。上面的例子的限定筛选条件是——找到div标签的直接子元素里包含2个span的那个元素。

  • children()：返回被选元素的直接子元素；

  • not()：方法与filter方法作用相反；

  • add()：将获取的元素添加到已经匹配的元素集合中，合并后返回元素集合；

  • index()：返回指定元素的索引；

  • each()：为每个匹配元素规定要运行的函数；

    • 语法：$(selector).each(function(index,element));
      • index：匹配的元素索引；
      • element：匹配的每个元素。
    • 参数可以省略不写。

  • animate()：jQuery动画方法，允许你创建自定义动画。animate默认提供队列功能，如果是多个动画的链式使用，则animate根据队列的先后顺序执行。

    • 语法：$(selector).animate({params},speed,callback);
      • params：给所选元素添加动画的属性（css属性）列表，必须；
      • speed：动画效果时常，取值可以是”slow”,”fast”或者ms，默认是normal，可选；
      • callback：动画完成后执行的回调函数，可以在该函数中再次使用动画效果，可选。

    注意：当使用 animate() 时，必须使用 Camel 标记法书写所有的属性名，比如，必须使用 paddingLeft 而不是 padding-left，使用 marginRight 而不是 margin-right，等等。

  • on()：给方法给被选元素或其子元素添加一个或多个事件。可运用于事件委托。

    • 语法：$(selector).on(event,childSelector,data,function);
      • event：规定添加到元素的一个或多个事件，可由空格隔开，给出多个；
      • childSelector：规定只能添加事件的子元素（常见于事件委托）；
      • data：给方法传递的额外数据，可以通过function里的e对象获取；
      • function：当事件发生时运行的函数。

  • off()：和on方法相反，注销元素身上由on方法添加的事件。

    • 语法：$(selector).off(event,selector,function,map);
      • event：移除的事件名；
      • selector：需要移除事件的当时通过on方法添加事件的元素；
      • function：事件发生时要执行的函数；
      • map：规定事件映射 ({event:function, event:function, …})，包含要添加到元素的一个或多个事件，以及当事件发生时运行的函数。

  • event.stopPropagation()：阻止事件冒泡到父元素（DOM树顶端）。

    注： 另外return false也可以阻止事件冒泡，但是要注意语句位置，一般是放于事件执行末尾。

  • event.preventDefault()：阻止元素的默认事件发生，如阻止a元素的打开链接的默认事件、阻止提交按钮点击时表单提交数据的默认事件……

    注意：可以使用isDefaultPrevented()方法检测元素是否调用了preventDefault方法。另外，return false也可以阻止元素的默认事件发生。

  • trigger()：触发被选所有元素的指定事件，且会触发默认事件和事件冒泡。

    • 语法：$(selector).trigger(event,param1,params……);
      • event：触发的事件，必须；
      • params：给触发方法传递的其他额外参数，可选。

  • triggerHandler()：与trigger方法类似——触发被选元素的指定事件，唯独不同的是该方法不会触发元素的默认事件和事件冒泡，且只触发所选元素中第一个元素的事件。语法与trigger一样。
