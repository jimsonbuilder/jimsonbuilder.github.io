二、git仓库操作相关命令

git作为分布式版本管理工具，与集中式版本管理工具（如svn）有不同：

  • git远远比svn速度要快；
  • 结构不同，一个是分布式管理，一个是集中式管理；
  • git分布式管理工具工作更灵活，svn集中式管理笨拙，需要联网才能工作。

两者最大的区别是：在git分布式模式下，每个开发者机器上都有一个服务器仓库，且开发者可以本地提交。

5、项目开发中的冲突问题解决

多人开发环境下很容易产生冲突，冲突的出现和解决方法如下：

如果多个项目的团队成员修改了同一个文件,那么就可能产生冲突：

  • 张三：修改lw.txt，然后git commit 提交到本地仓库，接着再git push到远程仓库；

  • 老王：修改lw.txt，然后也git commit 提交到本地仓库，接着再git push到远程仓库，此时报错(X)。

解决方法:

(1) 老王先更新获取最新的仓库信息：git pull；

(2) 老王git status查看哪个文件发生冲突，打开发生冲突的文件手动的进行处理(覆盖|合并)；

(3) 老王需要先add到暂缓区，然后commit到本地版本库，再推送到远程的仓库：git push；

(4) 张三需要更新获取最新的仓库信息保持同步：git pull。
